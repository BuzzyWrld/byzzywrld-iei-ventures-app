### Pattern files — what's distinctive

- **`messaging-voice.md`** — IEI's voice is built around a single named persona, the "Informed Sibling," who diagnoses rather than pitches, names a specific Enemy in every message, and walks away with comfort. Receipts (specific numbers, GSM, named mills, ROAS lifts) replace adjectives. Each brand gets a per-brand "Green-Light / Red-Light" word list as a secret-handshake vocabulary.
- **`audience-personas.md`** — Audiences are defined as **named individuals** ("Jordan," "Maya") moving from a hyper-specific Island A (current pain) to Island B (desired outcome), filtered through a binary Whale-vs-Vampire matrix that **rejects misfits aggressively** rather than broadening. 4-Stage Tribal Funnel governed by a Status Economy.
- **`offer-frameworks.md`** — Offers are framed as **named medical Prescriptions** ("The 90-Day Creative Refresh Engine," not "Premium Plan") following Diagnosis → Cost of Inaction → Prescription → Mechanism → First Win → Risk Reversal. Pricing is anchored against the **cost of doing nothing**, not against competitors. Discounting is forbidden; capped capacity ("2 clients per quarter") and tiered decoy/target/anchor structure preserve margin.

### `SKILL.md` — wired in 3 places

1. **New Step 0.5** ("Study the Exemplars") — model reads `INDEX.md`, picks 1–2 closest exemplars, reads them in full before drafting.
2. **Step 1** ("Build the Brand Foundation") — pattern files loaded at the right worksheet stages: `messaging-voice` for WS2/WS3, `audience-personas` for WS7/WS9/WS10, `offer-frameworks` for WS5/WS8.
3. **Reference Files** section at bottom — full inventory.

### Caveat (worth knowing)

Many of the 63 training-doc filenames don't match their contents (e.g. `Affiliate Program Performance & Logic.docx` actually contains audit-deck material). The distillation extracted concepts from wherever they actually lived. At some point the source files should be renamed.

## 2. Runtime — variant generators read exemplars

The brand-playbook skill is what Buzz uses inside Claude Code. But the IEI Ventures app's variant generators (`src/lib/variants/*.ts`) call Claude/DeepSeek directly via API — they don't run through the skill. So they need their own exemplar plumbing.

### New files

| File | Purpose |
|---|---|
| `src/lib/variants/exemplar.ts` | Cached file-read helper. Loads landing/pitch exemplars from `skills/brand-playbook/references/exemplars/`, frames them with "match QUALITY and STRUCTURAL PATTERNS, not specific copy or length" guidance, returns a string snippet to append to system prompts. |

### Modified files

| File | Change |
|---|---|
| `src/lib/variants/landing.ts` | (1) Loads `landingExemplar()` from the new helper. (2) **Refactored from one-call-three-variants to three parallel single-variant calls** — the old approach truncated JSON when the 47KB FamFit exemplar made the model verbose (response hit `maxTokens` mid-string). Each variant now gets its own 12K output budget. Includes explicit "exemplar = STYLE not LENGTH" instruction. |
| `src/lib/variants/pitch.ts` | Loads `pitchExemplar()` and appends to system prompt. Pitch is already a single-doc generation — no refactor needed. |

### Why the refactor matters

The first end-to-end demo bake produced 0 landing variants — the model signal-matched the FamFit exemplar's verbosity and overflowed `maxTokens: 8000`. Bumping to 16K just deferred the problem (next attempt overflowed at 43KB). The structural fix — parallel single-variant calls — keeps each request well within budget regardless of how rich the exemplar is.

### Token-cost impact

Per landing-page generation (3 variants, parallel):
- Before: 1 call × ~1K input + 8K output ≈ tiny
- After: 3 calls × (1K base + ~12K exemplar input) + 12K each output ≈ still ~$0.05 on Haiku, ~$0.01 on DeepSeek. Cache-friendly: the exemplar block is identical across the 3 calls so prompt-caching reuses it.

## 4. How this improves quality (one-line answer)

Claude doesn't "train" on these — it reads them at inference time as few-shot context, the way a junior designer reads past playbooks before drafting. Without exemplars, output drifts to generic LLM brand-deck filler. With them, the model pattern-matches to IEI's quality bar — vocabulary, section ordering, copy density, layout sophistication.
